# 文字モンスター育成RPG 要件定義書 v1.0

## 1. 概要

### 1.1 ゲームコンセプト

祖先モンスターの名前に含まれる文字を遺伝子として継承し、
プレイヤーが名前を決めながらモンスターを進化させる
**モンスター育成RPG**。

特徴： - 名前 = 遺伝子 - 家系システム - 文字順による能力差 -
勧誘による仲間化 - レベル到達による「ひらめき」システム

------------------------------------------------------------------------

## 2. プラットフォーム

  項目   内容

------------------------------------------------------------------------

## 3. 文字システム

### 3.1 使用文字

ゲームで使用する文字は以下。

-   基本漢字：約200文字（開発者定義）
-   追加レア文字：ひらめきによって解禁される文字

文字プールは共通管理とし、レア文字も同一プールに含める。

理由： - レア文字同士からさらにレア文字を生み出す設計を可能にする -
強力なレア文字（DQMの？？？系のような存在）を作れるようにする

------------------------------------------------------------------------

### 3.2 文字属性

各文字は以下の属性を持つ。

  属性      内容
  --------- ----------------------------------
  element   火 / 水 / 雷 / 土 / 風 / 光 / 闇
  role      攻撃 / 防御 / 速度 / 妨害 / 回復
  rarity    N / R / SR / SSR

------------------------------------------------------------------------

## 4. モンスター

### 4.1 名前

モンスターの名前ルール

-   1〜3文字
-   同一文字の重複は禁止
-   文字順は能力に影響する

例

炎鳥 ≠ 鳥炎

------------------------------------------------------------------------

### 4.2 能力計算

能力は **文字 × 位置重み**で決定する。

  文字位置   重み
  ---------- ---------------
  1文字目    攻撃寄り
  2文字目    防御/補助寄り
  3文字目    速度/特殊寄り

------------------------------------------------------------------------

## 5. 家系システム

参照できる祖先

直近3世代

進化時の命名では祖先の名前に含まれる文字を素材として使用できる。

------------------------------------------------------------------------

## 6. 仲間化（勧誘）

### 6.1 基本フロー

1.  戦闘（オート）
2.  勝利
3.  勧誘判定
4.  敵が仲間にして欲しいと申し出る
5.  プレイヤーが Yes / No を選択
6.  Yesで仲間加入

------------------------------------------------------------------------

### 6.2 勧誘率計算

最終勧誘率

finalRate = clamp(baseRate × rarityMultiplier × battleMultiplier)

#### rarityMultiplier

  レア度   倍率
  -------- ------
  N        1.2
  R        1.0
  SR       0.7
  SSR      0.4

#### battleMultiplier

  条件           倍率
  -------------- ------
  短ターン勝利   1.2
  無傷勝利       1.3
  属性有利勝利   1.1
  格下相手       0.8

------------------------------------------------------------------------

## 7. ひらめきシステム

### 発生条件

-   モンスター **Lv20到達**
-   **1モンスターにつき1回のみ**

------------------------------------------------------------------------

### 処理

モンスター名を入力として新文字を提示する。

例

炎鳥 → 鳳

------------------------------------------------------------------------

### プレイヤー選択

適用 / 見送り

#### 適用

-   新文字を命名素材として使用
-   名前を再設定
-   世代更新
-   新文字解禁

#### 見送り

-   新文字消失
-   再ひらめき無し

------------------------------------------------------------------------

## 8. ひらめきマッピング

ひらめきは固定マッピングで管理する。

例

炎鳥 → 鳳\
雷狼 → 神\
氷竜 → 魔

この仕組みにより

-   レア文字の連鎖
-   レア文字同士からさらにレア文字生成
-   強力なモンスター系統

などを実現できる。
